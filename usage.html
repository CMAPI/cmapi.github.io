  • Overview
  • CMAPI was orignaly developed to codify a reusable messagin spec for use in the Ozone Widget Framework (OWF) pub/sub mechinism. As CMAPI has matured, it has begun focussing on environemnt/runtime flexibility. Because CMAPI uses a well established concept, channel based pub /sub, the channels and data messages can really be used in any environment where this type of messing or eventing model is supported.
  • OWF
  • When using CMAPI in the Ozone Widget Framework it is recomeded to develop a widget according to the OWF widget developer's documentation included in the OWF distributions.
  • HTML Messaging
  • HTML messaging provides a means to use CMAPI as a communication protocal between applictations running is seperate browser windows. Although this is possible there are currently no known production ready implmentations of this capability available. Some prototypes have been developed whioch may be made avaialble to the community in the future.
  • Services
  • CMAPI can also be used a way to send messages between a ervice and a client, or between two services. This setup could be leveraged to accomplish multi-user collaboration between users in seperate locations.
