  • Overview
  • CMAPI was orignaly developed to codify a reusable messagin spec for use in the Ozone Widget Framework (OWF) pub/sub mechinism. As CMAPI has matured, it has begun focussing on environemnt/runtime flexibility. Because CMAPI uses a well established concept, channel based pub /sub, the channels and data messages can really be used in any environment where this type of messing or eventing model is supported.
  • OWF
  • To develop and codify a common API between applications and maps that will enable data from any program to interoperate with maps from any program.
  • HTML Messaging
  • The CMAPI specification is targeted at groups who need to communicate multiple application's data with a central map.

    In many cases multiple organizations have applications developed internally with different requirments that have a loose interoperability requirement with other data sources.


    CMAPI enables encapsulation of application business logic while allowing the user to interact with data on a common display.
