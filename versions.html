  • 1.2.0 - Release
  • CAMPI version 1.2.0 is the latest stable relase of CMAPI. The updates from version 1.1.1 include the additions of overlay level feature clustering, GeoJSON support, improved feature selection/deselection, OWF Widget intents, and more.
  • 1.3.0 - RC10
  • CMAPI 1.3.0 has been updated to "Release Candidate 10" as of 8/26/16

  • CMAPI 1.3.0 was moved to "Release Candidate" as of the 16 Oct 2014 CMAPI Technical Committee (TC) meeting, and as such should be stable enough for projects to implement. Version 1.3 is focused on adding user manipulation extensions to the API to enable more robust interaction between the the user, geospatial data, and the map. Some of the proposed capabilities in this version include end user drawing and editing of map features, querying of map features, batch processing of features, and feature-specific menus.

  • CMAPI v1.3 was moved to Release Candidate because the Technical Committee believes that the technical content of the specification is complete, and that updates from this point on should be limited to clarifications or grammatical fixes. At this point, it is reasonable for projects to start implementing this version of the specification. If there are any questions about how to interpret the wording of the specification, please email the CMAPI Google Group. This way, you get answers directly from the TC and other implementers - and the TC can then see where the wording in the specification needs to be improved.
